AVAILABLE TABLES


1. transactions: `lakshya-agenticai.retail.transaction_data`

   Column                  Type        Description
   ──────────────────────────────────────────────────────────────────────────────
   AMPERITY_ID             STRING      Unique customer identifier in UUID format (e.g. 4278f91d-44ba-3669…)
   DATASOURCE              STRING      Source system the transaction came from (e.g. CDT_POS_UIT = point of sale)
   ORDER_ID                STRING      Unique hashed identifier for the order

   LOYALTY_ID              STRING      Customer's loyalty program membership ID (null if not a loyalty member)
   CUSTOMER_ID             STRING      Hashed internal customer identifier (STRING, not INTEGER)
   AMPERITY_ID             STRING      Unique customer identifier in UUID format
   EMP_INDICATOR           BOOLEAN     TRUE if the transaction was made by an employee
   IS_INTERNATIONAL        STRING      Whether the order is international (STRING not BOOLEAN — probe values before filtering)
   PRODUCT_BRAND           STRING      Brand of the product (null in sample — probe before filtering)
   PRODUCT_CATEGORY        STRING      High-level product category (e.g. MEN'S, WOMENS — null in sample, probe before filtering)
   PRODUCT_CLASS           STRING      Product classification within category (null in sample)
   PRODUCT_COLOR           STRING      Color of the product (null in sample)
   PRODUCT_DEPARTMENT      STRING      Department the product belongs to (null in sample)
   PRODUCT_DESCRIPTION     STRING      Text description of the product (null in sample)
   PRODUCT_DIVISION        STRING      Division-level grouping above department (null in sample)

   PRODUCT_GROUP           STRING      Sub-grouping within product class (null in sample)

   ASSRT_LAYER2            STRING      Second level of assortment hierarchy (null in sample)

   ASSRT_LAYER4            STRING      Most granular assortment hierarchy level (null in sample)
   UNIT_COST               FLOAT       Cost per individual unit of the product

   FK_STORE_TXN            STRING      Foreign key linking to store transaction record

   PK                      STRING      Composite primary key combining store, product, interaction, and date fields

   Notes:
   • Revenue for a line item = ITEM_QUANTITY * ITEM_REVENUE (never use ITEM_REVENUE alone)
   • Many columns are null in sample data — always probe with SELECT DISTINCT or IS NOT NULL before filtering
   • IS_INTERNATIONAL is STRING, not BOOLEAN — check actual values before using in WHERE clause
   • LOYALTY_ID is null for non-loyalty customers; use IS NOT NULL / IS NULL to segment loyalty vs non-loyalty

   ORDER_FLAG                   INTEGER     1 if visitor completed a purchase during session, 0 otherwise
   SESSION_ATTRIBUTION_CHANNEL  STRING      Marketing channel that gets credit for session (e.g. social media organic, email, paid search)
   SESSION_ATTRIBUTION_CAMPAIGN STRING      Campaign name attributed to the session (e.g. Organic Social)
   IS_IN_APP_FLAG               BOOLEAN     TRUE if session originated from within the mobile app

   Notes:
   • CART_FLAG, CHECKOUT_FLAG, ORDER_FLAG are INTEGER (0 or 1) — use SUM(CART_FLAG) or AVG(CART_FLAG), NOT = TRUE
   • CUSTOMER_ID = -1 means anonymous visitor; filter with CUSTOMER_ID > 0 for identified users only
   • Funnel conversion: cart → checkout → order (each is a subset of the previous)

JOIN KEYS


These three tables do NOT share a common join key and should be queried independently.
  • AMPERITY_ID    → UUID format (transactions only)
  • SUBSCRIBERKEY  → email-address format (email only)
  • CUSTOMER_ID    → INTEGER in clickstream (STRING in transactions — different systems)
Cross-table JOINs will return 0 rows. Analyse each table on its own.
